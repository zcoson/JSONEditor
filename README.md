# JSON Editor 使用文档

## 简介

JSON Editor 是一款基于 React + Tauri 构建的跨平台桌面应用，专为 JSON 数据的查看、编辑和分析而设计。它提供了多列树形视图、实时编辑、数据过滤和统计等强大功能，帮助您高效处理复杂的 JSON 数据结构。

---

## 主要功能

### 📂 文件操作
- **打开文件**：通过菜单或快捷键打开 JSON 文件
- **保存文件**：支持保存到原文件或另存为新文件
- **拖拽打开**：直接拖拽 JSON 文件到窗口即可打开
- **双击打开**：在文件管理器中双击 JSON 文件直接启动应用并加载

### 🌳 多列树形视图
- **层级展开**：点击节点自动在新列中展开嵌套的 JSON 对象/数组
- **多列联动**：支持同时查看多个层级的数据结构
- **路径追踪**：每列顶部显示完整的数据路径
- **灵活布局**：支持水平和垂直两种布局模式

### ✏️ 数据编辑
- **实时编辑**：直接在编辑器中修改 JSON 值
- **智能输入**：根据数据类型自动切换输入控件（文本框、复选框、下拉框等）
- **表格模式**：数组中的对象自动以表格形式展示，便于批量编辑
- **撤销重置**：一键恢复到原始数据状态

### 🔍 预览与过滤
- **语法高亮**：Preview 模式下提供 JSON 语法高亮显示
- **数据过滤**：支持 JavaScript 表达式过滤和转换数据
- **统计函数**：内置 sum、avg、count、min、max、unique、groupBy 等统计函数
- **链式调用**：支持 filter、map、sort、slice 等数组方法链式调用

### 🎨 界面定制
- **字体缩放**：支持调整编辑器字体大小（10px - 24px）
- **面板调整**：拖拽分隔条自由调整各列宽度
- **布局切换**：水平/垂直布局一键切换

### 📋 快捷操作
- **复制数据**：支持格式化复制和压缩复制两种模式
- **粘贴导入**：从剪贴板粘贴 JSON 数据
- **键盘快捷键**：Cmd/Ctrl+S 快速保存

从 Releases 页面下载适合您系统的安装包：

```bash
npm run tauri:dev

# 生产构建
npm run tauri:build
```

## 使用指南

### 1. 加载 JSON 数据

#### 方法一：打开文件
- 点击菜单栏 `File` → `Open`
- 或使用快捷键 `Cmd/Ctrl+O`
- 选择 JSON 文件后自动加载

#### 方法二：拖拽文件
- 直接将 JSON 文件拖拽到应用窗口
- 松开鼠标即可加载

#### 方法三：粘贴数据
- 点击工具栏的 **Paste** 按钮
- 自动读取剪贴板中的 JSON 文本

#### 方法四：双击文件
- 在文件管理器中右键 JSON 文件
- 选择"用 JSON Editor 打开"
- 或直接双击（如果已设置为默认应用）

---

### 2. 浏览 JSON 结构

#### 树形视图导航
1. **左侧面板**显示 JSON 的树形结构
2. **点击任意节点**：
    - 基本类型（字符串、数字、布尔值）：在右侧编辑器中显示可编辑表单
    - 复杂类型（对象、数组）：在新列中展开子结构

3. **多列浏览**：
    - 每次点击嵌套的对象/数组都会在新列中展开
    - 列数无限制，可根据需要层层深入
    - 每列顶部显示当前数据的路径（如：`root → users → 0 → address`）

#### 示例
假设有以下 JSON：
```json
{
  "users": [
    {
      "name": "Alice",
      "address": {
        "city": "Beijing",
        "zip": "100000"
      }
    }
  ]
}
```

操作流程：
1. 点击 `users` → 新列显示数组内容
2. 点击数组索引 `0` → 新列显示第一个用户对象
3. 点击 `address` → 新列显示地址对象
4. 最终形成 4 列视图，清晰展示数据层级

---

### 3. 编辑 JSON 数据

#### 编辑基本类型

**字符串：**
- 显示为多行文本框，自动调整高度
- 直接输入新值，实时生效

**数字：**
- 显示为数字输入框
- 支持小数和负数

**布尔值：**
- 显示为复选框
- 勾选为 `true`，取消为 `false`

**null 值：**
- 显示为下拉框
- 可选择转换为 null、空字符串、0、true 或 false

#### 编辑对象

对象以键值对列表形式展示：
- **键名**：蓝色显示，固定宽度
- **值**：根据类型显示对应的编辑器
- 修改任意值后立即更新 JSON 结构

#### 编辑数组

**表格模式**（数组元素均为对象时自动启用）：
- 表头显示所有对象的键名
- 每行代表一个数组元素
- 可直接在单元格中编辑

**列表模式**（其他情况）：
- 显示索引和值的两列表格
- 逐行编辑每个元素

#### 示例
编辑用户信息：
```json
{
  "name": "Alice",  // 点击文本框直接修改
  "age": 25,         // 数字输入框调整年龄
  "active": true     // 复选框切换状态
}
```

---

### 4. 预览模式与数据过滤

#### 切换预览模式
1. 选中一个对象或数组节点
2. 点击右上角的 **Preview** 按钮
3. 显示语法高亮的 JSON 文本

#### 使用过滤器

在 Preview 模式下，可以使用 JavaScript 表达式过滤和转换数据。

**基本语法：**
- `root` 代表当前选中的节点
- 支持所有标准 JavaScript 数组和对象方法

**常用操作：**

```javascript
// 过滤数组 - 只保留 active 为 true 的项
root.filter(x => x.active)

// 提取字段 - 获取所有用户的名字
root.map(x => x.name)

// 链式调用 - 过滤后提取名字
root.filter(x => x.age > 18).map(x => x.name)

// 排序 - 按名字字母顺序
root.sort((a, b) => a.name.localeCompare(b.name))

// 截取 - 获取前 10 项
root.slice(0, 10)

// 获取对象键名
Object.keys(root)

// 获取键值对
Object.entries(root)
```

**统计函数：**

```javascript
// 求和 - 计算所有商品总价
sum(root.map(x => x.price))

// 平均值 - 计算平均分数
avg(root.map(x => x.score))

// 计数 - 统计元素数量
count(root)

// 最小值/最大值
min(root.map(x => x.age))
max(root.map(x => x.age))

// 去重 - 获取所有不重复的分类
unique(root.map(x => x.category))

// 分组 - 按分类字段分组
groupBy(root, 'category')

// 自定义分组 - 按状态函数分组
groupBy(root, x => x.status)
```

#### 过滤结果
- 过滤结果实时显示在预览区域
- 错误信息显示在过滤器下方（红色）
- 成功应用时显示结果摘要（绿色）
- 可以复制过滤后的数据

#### 示例场景

**场景 1：筛选活跃用户**
```javascript
root.users.filter(x => x.active && x.age >= 18)
```

**场景 2：计算订单总额**
```javascript
sum(root.orders.map(x => x.amount))
```

**场景 3：按城市分组统计**
```javascript
groupBy(root.users, 'city')
```

**场景 4：获取价格最高的 5 个商品**
```javascript
root.products.sort((a, b) => b.price - a.price).slice(0, 5)
```

### 5. 复制与导出

#### 复制数据
1. 选中要复制的节点
2. 点击右上角按钮：
    - **Copy**：复制格式化的 JSON（带缩进和换行）
    - **Compress**：复制压缩的 JSON（单行，无空格）

3. 复制成功后按钮显示 "Copied"，1.5 秒后恢复

#### 保存文件
- **快捷键**：`Cmd/Ctrl+S`
- **菜单**：`File` → `Save`
- 如果是新文件，会弹出保存对话框
- 如果已有文件路径，直接覆盖保存

---

### 6. 界面定制

#### 调整字体大小
- 点击工具栏的 **A-** / **A+** 按钮缩小/放大
- 点击中间的数字显示当前字号
- 范围：10px - 24px，默认 14px

#### 切换布局
- 点击工具栏最右侧的 **↔/↕** 按钮
- **水平布局**：列从左到右排列（默认）
- **垂直布局**：列从上到下排列

#### 调整列宽
- 将鼠标移到列之间的分隔条上
- 光标变为调整大小的图标
- 按住鼠标左键拖动即可调整

---

### 7. 其他功能

#### 清除数据
- 点击工具栏的 **Clear** 按钮
- 清空当前加载的所有数据
- 恢复到初始状态

#### 重置修改
- 点击工具栏的 **Reset** 按钮
- 丢弃所有编辑，恢复到最初加载的数据
- 仅在修改过数据后可用

#### 错误提示
- JSON 格式错误时，顶部显示红色错误条
- 过滤器语法错误时，过滤器下方显示错误信息
- 根据提示修正问题即可

## 常见问题

### Q: 为什么我的 JSON 无法加载？
A: 请检查 JSON 格式是否正确。可以使用在线 JSON 验证工具检查语法。应用会在顶部显示 "Invalid JSON" 错误提示。

### Q: 如何查看深层嵌套的数据？
A: 点击每一层的节点，会自动在新列中展开。列数没有限制，可以根据需要层层深入。

### Q: 编辑后如何撤销？
A: 目前没有逐步撤销功能，但可以点击 **Reset** 按钮恢复到最初加载的状态。

### Q: 过滤器支持哪些函数？
A: 支持所有标准 JavaScript 数组方法（filter、map、sort、slice 等），以及内置的统计函数（sum、avg、count、min、max、unique、groupBy）。

### Q: 可以编辑数组的结构吗（添加/删除元素）？
A: 当前版本仅支持修改现有元素的值，不支持添加或删除数组元素、对象属性。此功能将在后续版本中添加。

### Q: 如何处理超大 JSON 文件？
A: 应用针对性能进行了优化，但建议：
- 文件大小不超过 10MB
- 嵌套层级不超过 50 层
- 单层对象/数组元素不超过 10000 个

### Q: 数据会丢失吗？
A: 编辑过程中数据仅保存在内存中，建议定期使用 `Cmd/Ctrl+S` 保存。关闭应用前的未保存更改将会丢失。
